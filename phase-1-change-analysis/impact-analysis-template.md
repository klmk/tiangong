#### 争议类型说明

| 争议类型 | 说明 | 示例 |
|----------|------|------|
| **架构冲突** | 新功能要求与现有架构设计理念不一致 | 需要引入事件驱动，但现有系统是同步调用架构 |
| **模块冲突** | 实现新功能需要修改现有模块的核心逻辑 | 需要修改用户认证模块的核心验证流程 |
| **接口冲突** | 新功能需要改变现有接口的签名或行为 | 需要在返回值中增加字段，影响所有调用方 |
| **数据冲突** | 新功能需要改变现有数据模型或数据格式 | 需要在订单表中增加字段，影响所有订单相关查询 |
| **先例缺失** | 系统中没有类似功能的先例，方案存在不确定性 | 需要实现一种系统中从未有过的数据处理方式 |
