## 1. 评估概述

### 1.1 评估背景

本评估基于 Paoding 框架的 AI 能力评估体系，针对**客开（Customization）**场景进行专项扩展。客开场景对 AI 的要求与从零开发有显著差异，核心区别在于：

| 维度 | 从零开发 | 客开场景 |
|------|----------|----------|
| 代码理解 | 理解需求即可 | 必须理解已有代码 |
| 代码生成 | 按规范生成 | 必须适配已有风格 |
| 影响评估 | 关注新代码 | 必须评估对已有系统的影响 |
| 风险识别 | 新功能风险 | 必须识别回归风险 |

### 1.2 评估范围

| 评估维度 | 说明 |
|----------|------|
| 代码理解能力 | 对目标系统代码库的理解深度 |
| 变更影响评估能力 | 评估代码修改波及范围的能力 |
| 回归风险识别能力 | 识别修改可能破坏的现有功能的能力 |
| 遗留代码适配能力 | 理解和适配不规范/过时代码的能力 |
| 代码生成适配能力 | 按已有代码风格生成新代码的能力 |

### 1.3 评分标准

| 等级 | 分值 | 说明 |
|------|------|------|
| **A - 优秀** | 4-5分 | 完全满足客开需求，可独立完成 |
| **B - 良好** | 3-4分 | 基本满足，少量情况需要人工辅助 |
| **C - 一般** | 2-3分 | 部分满足，需要较多人工介入 |
| **D - 不足** | 1-2分 | 难以满足，需要大量人工补强或替代方案 |
